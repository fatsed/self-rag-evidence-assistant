Self-RAG stands for Self-Reflective Retrieval-Augmented Generation.

The main idea of Self-RAG is to improve the quality and factuality of language model outputs by combining retrieval, generation, and self-reflection.

Traditional RAG usually retrieves documents first and then generates an answer using those documents.

Self-RAG is different because it can decide whether retrieval is needed. If retrieval is useful, it retrieves relevant passages and generates an answer using them.

Self-RAG also critiques its own output. It checks whether the retrieved passage is relevant, whether the generated answer is supported by the evidence, and whether the answer is useful.

The main components of Self-RAG include retrieval, reflection, critique, retrieval tokens, and critique tokens.

A retrieval token helps decide whether external information should be retrieved.

Critique tokens help evaluate the quality of the retrieved evidence and the generated answer.

This project is a simplified version inspired by Self-RAG. It does not train a new language model. Instead, it uses an LLM API to simulate the main steps.
